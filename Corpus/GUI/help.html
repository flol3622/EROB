Documentation EROB

This documentation provides an overview of each parameter available in the GUI for the EROB simulation setup.

Basic Settings

Simulation

Simulation length (in days)

The duration of the simulation in days. Default is 365 days.

Year

The year to start the simulation. Default is 2021.

Building

Number of buildings

Specifies the number of buildings to simulate. Default is 1.

Dwelling type

Indicates the type of dwelling. Options include apartment or house.

Year of construction/renovation

The year the building was constructed or last renovated.

Number of bedrooms

Number of bedrooms in the building.

Ventilation system

Type of ventilation system in the building. Options are no system, exhaust ventilation, or balanced ventilation.

Household and Members

Number of persons

Total number of persons in the household.

Define gender

Option to specify the gender of each household member.

Define employment type

Option to define the employment type for each member (e.g., full-time, part-time, unemployed, retired, school, student).

Define bedroom assignment

Option to assign specific bedrooms to household members.

Heating

Setpoint settings

Predefined heating profiles. The profiles dictate temperatures for active, asleep, and absent states.

Heated rooms

Selection of rooms to be heated. Options include day zone, bathroom, and night zone.

Advanced Settings

Occupancy and Activity Profiles

Occupancy profile

Method to define occupancy profiles. Options include auto (default), custom, file, or no file.

Use file for activity profile

Option to use an external file for detailed activity profiles.

Appliances

Define present appliances

Selection of appliances present in the household. Options include:

  • Electricity: Printer, hob, oven, HiFi, iron, vacuum, kettle, freezer, refrigerator, microwave, fridge freezer, dishwasher, washing machine, tumble dryer, PC, TV, smartphone.
  • Water: Bath, shower, dishwasher, other flows.

Window Opening

Define window use habits

Option to specify window use habits for different rooms and seasons.

  • In general: Household habit in winter and seasonality coherence.
  • In detail: Custom habits for each room (living room, kitchen, bedroom, bathroom, leaving, sleeping) in winter and summer.

Additional Notes

When parameters are not defined by the user, the model will take a random value based on integrated probabilities.
